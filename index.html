Comandos Git

Segue os principais comandos Git, usados no gerenciamento do controle de versão de seus projetos de software.

init Inicializa um repositório Git em um diretório, criando um novo repositório vazio ou reutilizando um existente.
clone Clona um repositório Git existente para um novo diretório local. Isso é útil para obter uma cópia de um repositório remoto em sua máquina local para colaboração ou desenvolvimento.
branch Lista, cria ou exclui branches (ramificações) em um repositório Git. Branches permitem que você trabalhe em diferentes versões do seu código de forma isolada.
checkout Altera a branch atual ou restaura arquivos de um commit específico para o diretório de trabalho.
status Exibe o estado atual do diretório de trabalho e do índice (staging area), mostrando quais arquivos foram modificados, adicionados ou removidos.
diff Mostra as diferenças entre alterações em arquivos. Pode ser usado para ver as mudanças que foram feitas desde o último commit.
add Adiciona arquivos ao índice (staging area), preparando-os para serem incluídos no próximo commit.
commit Salva as alterações feitas no repositório, criando um novo ponto na história do projeto.
push Envia commits locais para um repositório remoto, atualizando-o com as alterações feitas.
pull Obtém alterações do repositório remoto e as mescla com o branch local atual.
revert Desfaz um commit específico, criando um novo commit que reverte as alterações introduzidas pelo commit original.
merge Combina alterações de diferentes branches em um único branch. Isso é comumente usado para incorporar o trabalho de um branch de recurso de volta ao branch principal.
stash Salva temporariamente alterações locais que não estão prontas para serem commitadas, permitindo que você as remova temporariamente do diretório de trabalho.
rm Remove arquivos do diretório de trabalho e os marca para exclusão no próximo commit.
config Configura opções do Git, como nome de usuário, email e preferências de merge.
reset Altera o estado do diretório de trabalho e do índice para corresponder a um commit específico.
remote Gerencia repositórios remotos, permitindo adicionar, remover ou renomear remotos.
fetch Obtém commits e referências de um repositório remoto, atualizando o repositório local sem mesclá-lo com o branch local.
show Exibe informações sobre commits, tags ou branches.
help Exibe a documentação de ajuda do Git para um comando específico.
rebase Move ou combina uma sequência de commits para um novo ponto base, reescrevendo a história do projeto.
blame Exibe quem modificou cada linha de um arquivo e em que commit essas modificações foram feitas.
tag Cria, lista ou exclui tags (marcadores) em commits específicos. As tags são usadas para marcar pontos específicos na história do projeto, como versões lançadas.

Configuração e acesso ao repositório GitHub

  1. Crie uma conta no GitHub: Se você ainda não tem uma, vá para o site do GitHub (https://github.com/) e crie uma conta gratuita.

  2. Instale o Git: O Git é um sistema de controle de versão que você precisará para interagir com o GitHub. Você pode baixá-lo e instalá-lo em seu sistema a partir do site oficial do Git (https://git-scm.com/).

  3. Configure o Git: Depois de instalar o Git, é necessário configurar seu nome de usuário e endereço de e-mail. Você pode fazer isso usando os seguintes comandos no terminal:
    4. git config --global user.name "Seu Nome"
    git config --global user.email "seu@email.com"

  4. Crie um repositório no GitHub: Faça login na sua conta do GitHub, clique no sinal de mais (+) no canto superior direito da página e selecione "Novo repositório". Siga as instruções para criar um novo repositório, dando um nome e uma descrição apropriados.

  5. Clonando um repositório: Para clonar um repositório do GitHub para o seu computador local, use o seguinte comando no terminal:
    6. git clone https://github.com/nome_do_usuario/nome_do_repositorio.git

  6. Adicionando arquivos: Depois de fazer alterações nos arquivos do repositório clonado em seu computador local, você pode adicionar essas alterações ao repositório usando os seguintes comandos:
    7. git add .
    git commit -m "Mensagem descrevendo as alterações"
    git push origin master

  7. Obtendo atualizações: Se outras pessoas fizerem alterações no repositório remoto e você deseja obter essas atualizações, use o seguinte comando:
    8. git pull origin master

Esses são apenas alguns dos comandos básicos para configurar e acessar um repositório no GitHub, existem muitos outros comandos e funcionalidades disponíveis.
